# ToonNotes Feature Announcement Email Templates

**Last Updated:** 2026-01-04
**Strategy Version:** v2 (AI Organization + AI Design)
**Use:** New feature launches, major updates

---

## Template Structure

Every feature announcement should follow this structure:

```
1. Subject line (curiosity + benefit)
2. Preview text (expands on subject)
3. Opening hook (pain point or excitement)
4. What's new (clear, concise)
5. Why it matters (benefit to user)
6. How to use it (simple steps)
7. CTA button
8. What's next (optional: roadmap tease)
9. P.S. (secondary CTA or personal note)
```

---

## Voice & Tone (v2)

| Do | Don't |
|----|-------|
| Lead with user benefit | Lead with feature name |
| Explain the "why" | Just list specs |
| Keep it scannable | Write walls of text |
| Sound excited but genuine | Sound like marketing |
| Connect to pain points | Assume they care automatically |

# SEGMENTATION GUIDELINES

## Who gets feature announcements:

| Feature Type | Audience |
|--------------|----------|
| Core features | All users |
| AI Organization features | All users |
| AI Design features | Users who've created 5+ notes |
| Pro features | Free users (conversion) + Pro users (value reinforcement) |
| Beta invites | Power users (20+ notes or 2+ boards) |
| Use case features | Segment by behavior if possible |

## Timing:

| Announcement Type | Best Send Time |
|-------------------|----------------|
| Major feature | Tuesday-Thursday, 10 AM |
| Minor update | Any weekday, morning |
| Pro features | After user hits free limit |
| Beta invites | Personalized timing |
