## Hero Section Overview

The hero is the first thing visitors see. It must:

1. **Stop the scroll** — Headline that resonates with pain
2. **Explain the product** — Clear value prop in seconds
3. **Create desire** — Show the outcome they want
4. **Drive action** — Clear, compelling CTA

**Target audience:** The Prolific Procrastinator
- Takes lots of notes
- Hates organizing
- Has tried other tools
- Wants effortless results

# VISUAL GUIDELINES

## Hero Image/Animation Options

### Option 1: Before/After Transformation
```
Left side: Chaos (messy notes, scattered saves, overwhelm)
Right side: Clarity (beautiful organized board)
Transition: Animated transformation between states
```

### Option 2: Product Demo
```
Screen recording or animation showing:
1. Note being captured
2. AI auto-labeling (labels appearing)
3. Final organized view
```

### Option 3: Board Showcase
```
Beautiful Knowledge Map or Trip Planner
Shows the end result users will get
Aspirational, aesthetic
```

### Option 4: Split Pillars
```
Two panels side by side:
- AI Organization (auto-labeling in action)
- AI Design (beautiful board output)
```

## Color Guidelines
```
Background: Clean white or subtle gradient
Accent: ToonNotes teal (#4C9C9B)
CTA: Coral (#FF6B6B) or teal
Text: Dark gray for readability
```

## Typography
```
Headline: Large, bold, high contrast
Subheadline: Medium weight, slightly smaller
Body: Regular weight, comfortable reading size
CTA: Bold, all caps or sentence case
```

# IMPLEMENTATION NOTES

## Above the Fold
Everything essential should be visible without scrolling:
- Headline
- Subheadline
- Primary CTA
- Hero visual

## Load Time
- Optimize hero image/animation
- Consider static image with lazy-loaded animation
- Keep CTA immediately clickable

## Accessibility
- High contrast text
- Alt text for images
- Keyboard-navigable CTA
- Screen reader friendly structure
